### 4. БРОНИРОВАНИЕ `/booking` — 5 ШАГОВ

**Прогресс-бар сверху:** 5 шагов с иконками, выполненные — зелёная галочка, активный — золотой

**ШАГ 1: Тип события**

- Карточки с иконками на выбор (одна карточка активна):
  - 📷 Фотосессия
  - 🎬 Видеосъёмка
  - 🎉 Мероприятие
  - 📢 Промоакция
  - 👗 Показ мод
  - 📺 Коммерческая реклама
  - 📋 Другое
- "Кнопка Далее →" активна только после выбора

**ШАГ 2: Выбор модели**

- Мини-каталог с фото и именами (те же карточки что в каталоге, но компактнее)
- Можно выбрать несколько моделей (checkbox-стиль)
- Фильтры: категория, город, только свободные
- Если пришли с /catalog → model_id предвыбран
- "Пропустить — не важно какая модель"

**ШАГ 3: Дата и бюджет**

- Date picker: дата события (запрет прошедших дат)
- Stepper: длительность в часах (−/+ кнопки, min 2, max 24, default 4)
- Поле: место проведения
- Поле: бюджет (необязательно)
- Textarea: комментарии к заявке
- Вверху мини-бар: выбранное событие + выбранные модели (превью)

**ШАГ 4: Ваши контакты**

- Имя (обязательно)
- Телефон (обязательно, маска +7 (**_) _**-**-**)
- Email (обязательно если включена настройка booking_require_email)
- Промокод (поле + кнопка "Применить" — live проверка из Supabase, показ скидки)

**ШАГ 5: Подтверждение**

- Сводка всех введённых данных (все поля)
- Кнопки "← Изменить" рядом с каждым блоком
- Чекбокс согласия с политикой
- Кнопка "✅ Отправить заявку" (POST /api/orders)
- Состояние загрузки (spinner на кнопке)

**Экран успеха (шаг 6 — после отправки):**

- Анимация ✅ (появляется)
- "Заявка №NM-2024-XXXX принята!"
- "Мы позвоним вам в течение 30 минут"
- Кнопки: "Проверить статус заявки" + "На главную"

**Черновик заявки:** сохраняется в sessionStorage, при перезагрузке предлагает продолжить

## ⚠️ ВАЖНЫЕ ПРАВИЛА

1. **Все тексты** (greeting, about, pricing, hours) — из Supabase `app_settings`, НЕ хардкодить
2. **Настройки-переключатели** влияют на UI: если `loyalty_enabled=0` — не показывать раздел лояльности
3. **Промокоды:** проверять через Supabase — expiry, max_uses, min_budget
4. **Файлы/фото:** загружать в Supabase Storage (bucket: `models`)
5. **JWT:** хранить в localStorage, передавать в заголовке `Authorization: Bearer ...`
6. **Все формы:** валидация на клиенте + сервере, показывать ошибки inline
7. **Loading states:** skeleton/spinner для КАЖДОГО async запроса
8. **Toast уведомления:** появляются сверху-справа, исчезают через 3 сек
9. **Пагинация:** бесконечная прокрутка ИЛИ "Загрузить ещё" для каталога
10. **URL синхронизация фильтров:** catalog?category=fashion&city=Москва — bookmarkable
