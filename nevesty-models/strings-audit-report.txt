RECOMMENDATIONS
---------------
1. PRIORITY: Replace the ~198 navigation button text literals in bot.js with
   STRINGS.btnBack, STRINGS.btnForward, etc. This eliminates the largest
   category of duplication and makes nav labels instantly translatable.

2. Use the getString(key, vars) helper for templates containing {name}, {id},
   {count}, etc. (e.g. STRINGS.adminModelAdded, STRINGS.broadcastSending).
   These are already parameterized in strings.js.

3. Consider splitting strings.js into domain modules (strings/booking.js,
   strings/admin.js, strings/catalog.js) once the file grows further — at
   179 keys it is still manageable as one flat object.

4. The Ukrainian strings scattered in bot.js ('🏠 Головне меню', '❌ Скасувати',
   '🔍 Пошук', '🔍 Розширений пошук', '✅ Ви вже залишали відгук...') should be
   moved into a separate STRINGS_UA object (or a uk locale key) to formalize
   multi-language support.

5. The utils/strings.js (RU object imported separately in bot.js line 5) should
   eventually be merged or aliased into this file to have a single source of truth.
