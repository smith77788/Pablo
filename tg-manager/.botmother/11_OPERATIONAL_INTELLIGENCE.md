# Operational Intelligence

BotMother should feel alive and aware.

## Live Operational Center

Show:
- active waves
- recent completions
- failed items
- visibility changes
- ecosystem health
- infrastructure alerts
- recommended fixes
- assets needing attention
- quick actions

Example:
“Publishing Wave completed: 48 channels updated. 3 failed. Retry failed?”

## Infrastructure pressure & health

System should understand:
- account pressure
- ecosystem load
- operation density
- queue pressure
- repeated patterns
- unstable assets
- timing pressure
- infrastructure fragmentation

Recommendations:
- rebalance
- slow down
- retry later
- exclude unstable assets
- fix permissions
- synchronize templates
- archive broken assets
- split large wave
- use another account pool

## Operation forecasting

Before execution, forecast:
- duration
- affected assets
- risk/safety
- skipped items
- missing permissions
- likely conflicts
- pressure level

## Infrastructure awareness

BotMother should understand:
- which accounts manage which channels
- which channels belong to which ecosystem
- which groups belong to which ecosystem
- which bots belong to which ecosystem
- which templates are applied
- which assets share accounts
- where duplicates exist
- where conflicts exist
- where fragmentation exists
- where permissions are missing
- where setup differs from target structure

The product must not treat assets as isolated rows.
