# Feature Priority Scoring

Do not implement impressive-sounding features just because they appear in the vision.

Before choosing work, score each candidate:

- user impact
- reusability
- architectural leverage
- mass-operation leverage
- safety improvement
- UX improvement
- implementation complexity
- maintenance cost
- risk of breaking existing flows

Prioritize:

1. Broken existing flows
2. Safety and reliability
3. Operation Engine foundations
4. Targeting and reports
5. Telegram-native UX clarity
6. Import and discovery
7. Templates / DNA / drift
8. Visibility and intelligence
9. Billing and governance
10. Advanced automation

Avoid:

- isolated niche features
- cosmetic complexity
- dead menus
- fake placeholders
- features without preview/retry/reporting
- advanced systems on unstable foundations
