# Execution Protocol

## Absolute repository rules

Do NOT:
- rebuild the app from zero
- create a new app
- destroy existing architecture
- remove working systems
- replace working flows without reason
- make the product web-first
- create generic SaaS patterns
- create random Telegram button dumps
- create fake features
- create dead menu items
- write code before repository audit

Do:
- inspect the repository first
- preserve working functionality
- merge this memory into existing project context files
- improve incrementally
- keep backward compatibility where possible
- improve Telegram-native UX
- improve mass workflows
- improve previews, retries, reports
- improve explanations and structure

## Mandatory workflow before coding

Before writing code:

1. Inspect repository structure.
2. Detect stack, frameworks, package manager, backend/frontend boundaries.
3. Read existing README / CLAUDE / AGENTS / project context files.
4. Identify implemented features.
5. Identify incomplete features.
6. Identify broken features.
7. Identify duplicated logic.
8. Identify scattered menus/buttons.
9. Compare current project with the AI memory files.
10. Produce gap analysis.
11. Produce incremental implementation roadmap.
12. Select the smallest high-impact task.
13. Only then implement.

## After coding

Always:
- run lint if available
- run typecheck if available
- run tests if available
- report any failures
- summarize changed files
- summarize risks
- summarize remaining work

## Priority system

Priority 0:
- repository audit
- architecture understanding
- persistent context merge

Priority 1:
- fix broken systems
- make existing features actually work
- remove dead/fake flows
- improve Telegram-native UX
- improve menu structure
- add explanations
- add previews/retries/reports to existing mass actions

Priority 2:
- extend account/bot/channel/group management
- improve factories
- improve templates
- improve import center
- improve ecosystems
- improve universal action engine

Priority 3:
- visibility engine
- regional scaling
- infrastructure memory
- AI assistant
- advanced analytics
- topology
- operational forecasting

Avoid building huge abstract systems before improving real user workflows.
