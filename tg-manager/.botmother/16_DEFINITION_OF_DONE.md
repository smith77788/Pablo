# Definition of Done

A feature is NOT complete unless it has:

- correct placement in product structure
- simple explanation
- working input flow
- preview if needed
- confirmation if needed
- execution logic
- success state
- failure state
- retry path if applicable
- report if mass action
- history/audit integration
- Telegram-native UX
- no dead buttons
- no fake behavior
- no confusing labels
- no duplicate feature path

If not connected yet, clearly mark:
- Coming soon
or
- Not connected yet

Never pretend a feature works if it does not.
