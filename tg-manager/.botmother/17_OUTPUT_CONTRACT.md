# Output Contract

After work, always report:

1. What was analyzed.
2. What already existed.
3. What was changed.
4. Why it was changed.
5. Files changed.
6. Risks introduced.
7. Remaining limitations.
8. Suggested next steps.
9. Commands run:
   - lint
   - typecheck
   - tests
10. Whether anything failed.

If architecture changed:
- explain why
- explain impact
- explain migration risk

If a feature was added:
- explain where it lives
- explain how to use it
- explain what is still missing
