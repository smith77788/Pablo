# Telegram UX Governance

This strengthens `04_TELEGRAM_NATIVE_UX.md`.

BotMother should feel powerful but not chaotic.

## Every screen must answer

- Where am I?
- What is this for?
- What can I do now?
- What happens next?
- How do I go back?
- How do I get help?

## Menu rules

Every menu should have:

- short title
- one-sentence purpose
- grouped actions
- Back button
- Help button
- status summary where useful
- Advanced section when needed

Do not show too many unrelated buttons at once.

## Prefer

- progressive disclosure
- guided flows
- smart defaults
- search
- filters
- quick actions
- pagination
- action palette
- contextual recommendations

## Never

- create dead-end flows
- expose raw internal complexity unnecessarily
- use confusing labels
- create 30-button dumps
- hide operation status
- start risky mass actions without preview and confirmation
