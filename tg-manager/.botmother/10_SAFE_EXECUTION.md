# Safe Execution & Natural Operational Behavior

## Core principle

Infragram must coordinate operations in a sustainable, balanced, stable way.

The system should not behave like a rigid script.

## Prioritize

- sustainable execution
- balanced pacing
- infrastructure stability
- account preservation
- safe operation distribution
- realistic scheduling

## System should

- distribute actions over time
- avoid rigid repetitive timing
- avoid overloaded execution
- avoid simultaneous bursts
- support timezone-aware scheduling
- support regional timing
- stagger large operations
- balance account pools
- monitor infrastructure pressure
- adjust timing when needed

## User-facing concepts

Use:
- Safe Timing
- Execution Forecast
- Estimated Completion
- Balanced Execution
- Safety Score
- Timing Adjusted

Avoid making low-level platform limitation terminology a main UX concept.

## If hard platform limits happen

- treat as exceptional
- pause affected part
- adjust future timing
- rebalance execution
- explain simply
- log internally

Never:
- bypass restrictions
- force aggressive retries
- ignore limits
- implement evasion logic
