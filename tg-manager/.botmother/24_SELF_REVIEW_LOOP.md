# Self-Review Loop

Before finalizing code changes, review:

1. Did I inspect the repository first?
2. Did I preserve existing architecture?
3. Did I duplicate existing logic?
4. Did I create unnecessary new abstractions?
5. Did I improve reusable infrastructure?
6. Can this feature work in mass-operation context?
7. Does it support preview / confirmation / retry / reporting where needed?
8. Is Telegram UX clear?
9. Are errors explained in human language?
10. Are secrets protected?
11. Are operations auditable?
12. Did I run available checks?
13. Did I update docs?
14. Did I clearly report risks and remaining work?

If the answer is weak, improve the change before final response.
